## Method

1. **POTX renders** — opened `SAP_Corp.potx` (renamed to `.pptx` via Content-Type patch, see [audit/render_potx.py](render_potx.py)) in PowerPoint via COM automation, exported each layout to PNG at 1920×1080. 10 specimen-backed renders are high-signal (real demo content from POTX slides 1-20); 25 synthesized renders show only the layout chrome (placeholders, no user content).
2. **v0.3 renders deferred** — attempted via Playwright against `audit/audit-deck.md` running in `slidev` dev mode, but Vite/Slidev's lazy slide compilation crashed esbuild's transform service repeatedly, leaving slides as 1280×720 white blanks. Static-build path also failed (production build issues — see Findings B-1 and B-2). For this report, the **user's pre-flagged screenshots** serve as v0.3 references for those layouts; the rest are described from the v0.3 implementation source.

## Severity scoring

- 🔴 **Broken** — layout doesn't render correctly OR is so far from POTX that authors would actively avoid it
- 🟠 **Wrong** — readable but visually doesn't match POTX brand (wrong composition, wrong typography, missing elements)
- 🟡 **Off** — close to POTX but with a specific named gap (e.g. wrong color tone, missing partner-logo box)
- 🟢 **OK** — visually matches POTX or close enough that the gap is cosmetic

## Pre-flagged user issues (top of priority)

These came directly from user feedback before the audit. They take precedence in the v0.4 fix plan because they're already known broken.

### P-1 🔴 RipplePattern decoration size + placement (cover-a)

**User screenshot showing v0.3.0 (before v0.3.1 fix):**
The Ripple appeared as a 30%-tall full-bleed wallpaper behind the cover title.

**User screenshot after v0.3.1 fix:**
The Ripple is a tiny letterboxed postage-stamp at the bottom-center of the slide.

**POTX cover-a:**

![POTX cover-a](potx-renders/cover-a.png)

POTX shows the slide split 50/50: left half white with title + Name/Date + dashed-border partner-logo box, right half **solid Blue 11 (#00144A)** with **Anvil Grid Pattern** (small Flat Anvils tiling at low density, fading toward the center) as decoration. The "Ripple Pattern" name in our spec was a misread of the brand portal — this is actually the Anvil Grid Pattern, used as a **right-half column**, not a footer band.

**Root cause:** my v0.3 spec called the wrong decoration ("Ripple Pattern" instead of "Anvil Grid"), placed it in the wrong location (footer band instead of right-half column), and used `object-fit: cover` then `contain` — both wrong because the pattern should **tile**, not stretch or fit.

**v0.4 fix scope:** Substantial. RipplePattern needs to become "right-half AnvilGrid decoration" with `background-image: url(...) / repeat-y` that tiles the SVG vertically across a 50%-wide right column. Cover layout needs to split content L/R explicitly. The current `<RipplePattern>` component should be renamed `<AnvilGridDecoration>` (or kept and re-implemented).

### P-2 🟠 Slide-overview close button positioned outside viewport

**Source:** Slidev built-in (`pages/overview.vue`), not our theme. Pressing `o` opens a fullscreen-ish panel that uses `h-[calc(var(--vh,1vh)*100)]` with the close `×` button at `top-4 right-4`. On displays where the slide canvas extends below the bottom of the browser chrome (or where browser zoom is non-100%), the top-4/right-4 close button can sit above the visible viewport.

**Current workaround:** `o` to dismiss, OR `Escape` (in some Slidev builds), OR scroll up.

**v0.4 fix scope:** Small. Add a global CSS override in `theme/styles/index.css` that pins Slidev's close button into the visible viewport with `position: sticky` or a min-top guard. Specifically:

```css
.slidev-quick-overview .icon-button[title="Close"] {
  position: sticky;
  top: 1rem;
  z-index: 100;
}
```

We should also document the `o` keybinding more prominently in CONTENT-GUIDE.md (only currently in README's keyboard shortcuts table).

### P-3 🟡 Bio avatar overlaps name text (single-presenter mode)

**User screenshot:** The `<ui5-avatar size="L">` (80px) intrudes into the `<ui5-card-header>` title-text region, causing "TJ" initials to visually run into "Thomas Jung".

**Root cause:** UI5 web component sizing mismatch. `ui5-card-header`'s avatar slot allocates ~64px; `size="L"` is 80px. Was a v0.2 latent bug carried forward into v0.3.

**v0.4 fix scope:** Trivial. Change `size="L"` → `size="M"` in [theme/components/Bio.vue:45](../theme/components/Bio.vue) — single-character fix.

### P-4 🟠 Agenda layout looks generic, not POTX-brand (agenda-b)

**User screenshot:** Plain ordered list with grey hairlines, light-grey description text. Looks like a todo-list, not SAP-branded.

**POTX agenda-b:**

![POTX agenda-b](potx-renders/agenda-b.png)

POTX shows: **bold serial numbers** `01, 02, 03, 04, 05` in heavy black sans (likely 72-Bold), **black bold "Agenda Sample Text"** labels (not muted grey), light-grey description text below each label, light-blue hairlines between rows, and **3-4 large blue Flat Anvil shapes filling the right ~25% of the slide** as decoration.

Our v0.3:
- Numbers blue-7 (not black-bold) ❌
- Item titles light-grey (not black-bold) ❌
- No anvil decoration on the right ❌
- Body text styling matches roughly ✓
- Hairlines between rows roughly match ✓

**v0.4 fix scope:** Substantial. Rebuild agenda layout to match POTX composition.

### P-5 🔴 Bio team mode broken on multiple axes

**User screenshot:** "Meet our team" with `<Bio :people="[...]">` shows:
- Anvil pattern stretching as huge slanted bars across the entire slide background (wrong scale/orientation)
- Photo cards positioned with top-half on dark background and bottom-half white (correct intent)
- Cards wrap to a second row mid-pattern (Ethan's card on row 2)
- QR codes clipped at the bottom of cards
- Names use 72-Bold (correct)
- No header band — anvil pattern fills *everything*, cards float on top

**POTX `Meet our team` (slide 14, would render from agenda-b layout but is a bespoke composition):** Tight, low-density Anvil Grid as a **header band** (top ~30%), 4 photo cards in **single row** breaking the band's bottom edge, names in 72-Bold black, roles in grey, **white background below the band** for the bottom half of cards + QR codes. Cards never wrap because the layout sizes them to fit single-row at the standard 4-person team size.

**v0.4 fix scope:** Substantial. Redesign Bio team mode:
- AnvilGrid as fixed top band (~30%), not full-bleed
- Cards `grid-template-columns: repeat(4, 1fr)` (not auto-fit, so they never wrap at the design size)
- Card height computed so QR + role + name fit without clipping
- Card top edge anchored within the band (`position: absolute; top: 20%`) so they break the band correctly

## ~~Discovered during audit~~ — RETRACTED

The two findings below were **false positives**. They surfaced when I ran `npx slidev build audit/audit-deck.md` against my throwaway audit deck, but `npm run build` against the actual repo's `slides.md` succeeds in 3.27s and produces a working 9.7 MB `dist/`. The deployed GitHub Pages site has been building correctly. The audit-deck failures were specific to its malformed structure (back-to-back frontmatter blocks with unusual cwd resolution), not to any v0.3 component bug.

### ~~B-1 Production build fails on `RipplePattern.vue`'s `?raw` import~~ — FALSE POSITIVE

The `?raw` import path *would* fail Rollup if Slidev didn't pass `inline=false` everywhere in the gallery deck. In practice, the regular `npm run build` chunks `RipplePattern` into a separate file (`dist/assets/RipplePattern-*.js`) and the dynamic `?raw` import inside `watchEffect` is treated as a deferred async import, so Rollup creates the right chunk and the build succeeds.

Latent gotcha for the future: if a deck *uses* `:inline="true"`, the runtime fetch may need different handling at build time. We don't currently use it, so no fix needed in v0.4.

### ~~B-2 Production build fails on `import.meta.glob` YAML data sources~~ — FALSE POSITIVE

The same `npm run build` resolves all `import.meta.glob('/event.yaml', ...)` calls correctly via Vite's path resolver. The audit-deck failure was a path-resolution edge case from running a deck inside `audit/` where the relative path to the theme broke YAML resolution.
