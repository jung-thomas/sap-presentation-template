# v0.4.3 — Text-with-icons + Table layouts

**Status:** Draft for review
**Date:** 2026-06-16
**Author:** Thomas Jung (with Claude assistance)
**Supersedes:** none — adds new layouts to v0.4.2.3
**POTX source:** `SAP_Corp.potx` → `ppt/slides/slide9.xml` (Text with icons), `ppt/slides/slide11.xml` (Table – four columns)

## 1. Summary

v0.4.3 ships **two new content layouts** extracted from the POTX:

1. **`text-with-icons`** — 3×2 grid of icon + title + description + optional "Learn more" link, mirroring POTX slide 9. Auto-fits to 3, 4, 5, or 6 items based on the content count.
2. **`table`** — 4-column structured table with branded header (SAP-blue-7 fill, white text) and light-blue (sap-blue-4) cell borders, mirroring POTX slide 11. Cells accept Slidev-native markdown table syntax.

Both are defined entirely from POTX measurements (see appendix).

## 2. Goals

1. **Faithful POTX reproduction** — every measured value (positions, sizes, colors, fonts) within ±1% of POTX.
2. **Speaker-friendly authoring** — frontmatter-only API for `text-with-icons`; markdown body for `table`.
3. **Reuse existing primitives** — `<ClassificationFooter>`, SAP brand tokens (`--sap-blue-*`, `--sap-font-family`). No new shared components needed unless an icon component is missing.
4. **Icon support via SAP web components** — use `@ui5/webcomponents-icons` (already a project dependency or addable as one). Authors reference icons by name string (e.g., `lightbulb`).

## 3. Non-goals

- Roadmap/timeline layout (slide 10) — deferred to v0.5+ (high-effort connector logic).
- Data points layout (slide 8), diagram-with-icons (slide 12), team-single (slide 13) — deferred.
- Custom POTX icon extraction — using the existing UI5 icon set instead. If a brand-specific icon is missing later, swap individual icons by overriding the icon name.
- Internationalization — `en-US` only.

## 4. `text-with-icons` layout

### 4.1 POTX measurements (slide 9, 1920×1080)

| Element | Value |
|---|---|
| Title position | top-left, ~4% inset, ~24pt |
| Cell origin (col 1, row 1) | left: 4.1%, top: 22.2% |
| Cell width | 29.2% of slide |
| Cell height | ~28% of slide (icon block + 3 text rows + link) |
| Column count | **3** (3 across × 2 down = 6 items) |
| Column gap | left edges at 4.1%, 35.4%, 66.6% — column gap ≈ 2% |
| Row gap | row 1 top 22.2%, row 2 top 60.6% — row gap ≈ 9% |
| Icon size | 4.1% × 7.3% (= 79×79 px on 1920×1080) |
| Icon-to-title gap | 8.7% of slide height (= ~94 px) |
| Title font | 16pt 72 Brand Medium, near-black |
| Description font | 14pt 72 Brand, black, 17.3% height block |
| Link "Learn more ›" | 14pt, color `#0070F2` (= sap-blue-7), 10.6% × 3.1% block |

### 4.2 Architecture

Single layout file `theme/layouts/text-with-icons.vue`:

- Root `<div class="text-with-icons">`
- `<h1 class="text-with-icons-title">{{ fm.title }}</h1>` (optional — POTX has a small "Text with icons" eyebrow but real decks usually replace it with a section title; render only when `title:` set)
- `<div class="text-with-icons-grid" :class="`cols-${columnCount}`">`
  - `<div v-for="item in items" class="text-with-icons-cell">`
    - `<ui5-icon :name="item.icon" class="text-with-icons-icon" />`
    - `<h3 class="text-with-icons-cell-title">{{ item.title }}</h3>`
    - `<p v-if="item.description" class="text-with-icons-cell-desc">{{ item.description }}</p>`
    - `<a v-if="item.link" :href="item.link.url" class="text-with-icons-cell-link">{{ item.link.text }} ›</a>`
- `<ClassificationFooter />`

`columnCount` is derived from `items.length`:

| items | cols |
|---|---|
| 1–3 | 3 |
| 4 | 4 (or 2×2 — TBD; pick whichever reads cleaner; spec says 4) |
| 5 | 5 |
| 6 | 3 (POTX-faithful: 3×2) |
| 7+ | 3 (3×N rows) |

Yes, 6 items → 3 cols × 2 rows by default, matching POTX. 4 and 5 items render single-row to avoid orphan rows.

### 4.3 Author API

```yaml
---
layout: text-with-icons
title: Why CAP?
items:
  - icon: lightbulb
    title: Productivity
    description: Less boilerplate, more declarative modeling.
    link:
      text: Learn more
      url: https://cap.cloud.sap
  - icon: people
    title: Reuse
    description: Composable services and entities.
  # ...up to 6 items
classification: PUBLIC
---
```

Required fields per item: `icon` (string, UI5 icon name) and `title` (string).
Optional per item: `description` (string), `link.text` + `link.url` (object).

### 4.4 Icons

Strategy: `@ui5/webcomponents-icons`. This package ships a large bundle of SAP-branded icons by name (e.g., `lightbulb`, `people-connected`, `target-group`).

If the package isn't already a dependency, add it. Wire it into `theme/setup/main.ts` so the `<ui5-icon>` web component is available globally to slides.

Icon color defaults to `currentColor`; the cell sets `color: var(--sap-blue-7)` so icons render in SAP brand blue. Authors can override per-cell via `iconColor:` in the item.

## 5. `table` layout

### 5.1 POTX measurements (slide 11, 1920×1080)

| Element | Value |
|---|---|
| Table position | left: 4.1%, top: 22.5% |
| Table size | 91.8% × 48.4% of slide |
| Column count | 4 (equal width 22.95% each = ~440 px) |
| Header row | height 8.1%, fill `#0070F2` (sap-blue-7), text `#FFFFFF`, 18pt 72 Brand Medium, center-aligned |
| Body rows | height 8.1% each (~87 px), text `#000000`, 14pt 72 Brand, center-aligned |
| Cell border | 1pt `#89D1FF` (sap-blue-4) on right + bottom edges |
| Cell padding | ~0.12" left/right (= ~14 px), ~0.06" top/bottom (= ~7 px) |

### 5.2 Architecture

Single layout file `theme/layouts/table.vue`:

- Root `<div class="layout table-layout">`
- `<h1 v-if="fm.title">{{ fm.title }}</h1>` (optional title — POTX has one but speakers can omit)
- `<div class="table-frame"><slot /></div>` — slot accepts the markdown body
- `<ClassificationFooter />`

The slot renders Slidev's standard markdown table → HTML `<table>`. Scoped CSS in `table.vue` styles `.table-frame > table` to match POTX:

```css
.table-frame :deep(table) {
  width: 100%;
  border-collapse: collapse;
  font-family: var(--sap-font-family);
}
.table-frame :deep(thead th) {
  background: var(--sap-blue-7);
  color: #ffffff;
  font-family: var(--sap-font-family-bold, var(--sap-font-major));
  font-weight: 600;
  font-size: 1.125rem;        /* 18pt ≈ 1.125rem */
  text-align: center;
  padding: 0.5rem 0.875rem;
  border: 1px solid var(--sap-blue-7);
}
.table-frame :deep(tbody td) {
  font-size: 0.875rem;         /* 14pt ≈ 0.875rem */
  text-align: center;
  padding: 0.5rem 0.875rem;
  border: 1px solid var(--sap-blue-4);
  color: var(--sap-text-primary, #000);
}
```

### 5.3 Author API

```yaml
---
layout: table
title: CAP vs. ABAP
classification: PUBLIC
---

| Capability | CAP | ABAP Cloud | RAP |
|------------|-----|------------|-----|
| Service modeling | CDS | CDS | CDS-based |
| Custom logic | Node.js / Java | ABAP | ABAP |
| Deployment | Cloud Foundry / Kyma | BTP ABAP env | S/4HANA |
| Best for | Greenfield apps | S/4 extensions | RAP-native apps |
```

The 4-column count is enforced by the **POTX visual style** — the layout doesn't reject 3- or 5-column markdown tables; they just won't have the POTX "exactly 4 equal columns" proportion. This is a soft constraint. Authors are expected to use 4 columns for the canonical POTX look.

## 7. Risks

| Risk | Impact | Mitigation |
|---|---|---|
| `@ui5/webcomponents-icons` package isn't already in `package.json`; adding it bloats bundle | Med | Add as a regular dep (it's small, ~50KB gzipped for the icon font). Cite the SAP-brand alignment as justification. |
| Markdown tables don't render thead/tbody distinction reliably across all Slidev markdown plugins | Low | Verify in dev server during impl. If header row isn't auto-distinguished, fall back to a frontmatter `headers: [...]` + `rows: [[...]]` schema with custom rendering. |
| 4-item count for text-with-icons is ambiguous: 4 across (1×4 row) or 2×2 grid? | Low | Spec says **1×4 single row** — keeps consistent "3-6 across" model. If 2×2 reads better in practice, adjust during impl. |
| POTX uses 72 Brand Medium / 72 Brand fonts; our `--sap-font-family` token may not load these in dev | Low | Use the existing token; POTX ships these fonts but the project's fallback chain (system fonts) is acceptable for development and CI. |

## 8. Open questions

All deferred to implementation phase, not blocking spec approval:

1. **`@ui5/webcomponents-icons` registration mechanism** — does `theme/setup/main.ts` already wire UI5 web components, or do we need to add the package + register the `<ui5-icon>` custom element? Verify and add the registration during impl.
2. **Icon name mapping** — speakers will want intuitive names (`lightbulb`, `people`, `target`). UI5 names are sometimes more verbose (`bo-lightbulb`, `people-connected`). Decide during impl whether to provide a thin alias map or document UI5's exact names.
3. **Default item count display** — if the author provides `items: [...]` with 0 items, the layout should render the title + an empty grid silently (or with a helpful warning in dev mode).
