## 1. Summary

v0.4.1 ships four things:

1. **AnvilGridDecoration.vue restoration** — already on the `hotfix/v0.4.0.1` branch (PR #8); merging to main is a prerequisite for this release.
2. **P-2 Slidev "Goto..." dialog fix** — bounds viewport + injects working close × button (already committed: `d0df851`).
3. **P-4 Agenda layout rebuild** — replace v0.3's plain `<ol>` with the POTX agenda-b composition: 67% left content (5 numbered items, black-bold labels, sap-blue-6 hairlines) + 33% right anvil-grid column.
4. **P-5 Bio team mode rebuild** — replace v0.3's full-bleed AnvilGrid + auto-fit cards with the POTX slide-14 composition: 50% top anvil-grid band + 4 cards in a fixed grid anchored to break the band edge.

All design values are measured from POTX renders, not estimated. Every dimension below is followed by `(POTX: x of W)` showing the source measurement.

## 2. Goals

1. **Faithfully reproduce POTX agenda-b** — every measurable value matches POTX within ±1% slide-relative tolerance.

4. **Backward compatibility** — v0.3 author API for both layouts continues to work; the visual changes apply automatically.

### 4.2 Architecture

**Single layout file** (`theme/layouts/agenda.vue`) composing:

- **Left content region** — 67% width: `<AgendaTitle>` + `<AgendaList>` (one row per item: number + label + description, with hairline below each except the last).
- **Right decoration column** — 33% width: `<AnvilGridDecoration bg="var(--sap-blue-10)" color="var(--sap-blue-6)" />` (the v0.4.0 primitive, just placed in a different region).

No new components needed beyond a tiny `<AgendaItem>` if it makes the layout file cleaner. Reuse `<ClassificationFooter>` as in v0.4.0.

### 4.3 Author API

**Unchanged from v0.3:**

```yaml
---
layout: agenda
title: Agenda
items:
  - title: Foundations of CAP
    description: Optional description text  # NEW: rendered below title
  - title: The data modelling layer
    description: Optional description text
  - ...
classification: PUBLIC
---
```

**Additions:**

- `description` field on each item is now first-class (matches POTX's "Add a short description of Agenda item..." placeholder text). If omitted, the description row is suppressed.
- Existing v0.3 `agenda` config object (`toc`, `showSubsections`, `dividers`, `sectionNumbers`, `slideNumbers`) is preserved for backward compatibility but `sectionNumbers` is now hard-coded `true` (POTX always shows them).

**Removed:** the `variant: a` / `variant: b` distinction. POTX agenda-a is just a blank slide; the new layout always renders the agenda-b composition. The `agenda-layout--b` CSS class is dropped.

### 4.4 Testing

- Unit test on a parameterized `<AgendaList>` with 1–8 items: row positions, hairline placement, label/description rendering all assert match.
- Visual regression baseline rebaselined for the agenda slide in the gallery deck.

### 5.2 Architecture

**Single component update** (`theme/components/Bio.vue`'s team-mode branch). The team-mode template composes:

- `<AnvilGridDecoration bg="var(--sap-blue-10)" color="var(--sap-blue-6)" class="bio-band" />` constrained to top 50% via scoped CSS (`.bio-band { position: absolute; top: 0; left: 0; right: 0; height: 50%; }`).
- `<h2 class="bio-title">Meet our team</h2>` overlay anchored top-left of the band, white text.
- `<div class="bio-cards">` containing exactly 4 `<article class="bio-card">` elements via a fixed `grid-template-columns: repeat(4, 1fr)` with 47px gap and 80px / 81px outer margins. The grid is positioned at `top: 24.4%; bottom: 12%;` so cards naturally overlap the band by 25.6%.

Each card renders: photo (top, 4:5 portrait crop) → name (72-Bold black) → role (grey) → QR code (fixed 64×64 bottom-right of card text region).

Replaces v0.3's `<AnvilGrid>` (the multi-cell grid component with `position: absolute; inset: 0`) and the `auto-fit, minmax(180px, 1fr)` card grid.

### 5.3 Author API

**Unchanged from v0.3:**

```html
<Bio :people="[
  { name: 'Nina Thompson', role: 'Data Analyst', photo: '/team/nina.jpg', qr: 'https://...' },
  { name: 'Marcus Bennett', role: 'CSM', photo: '/team/marcus.jpg', qr: 'https://...' },
  { name: 'Sofia Nguyen', role: 'QA Engineer', photo: '/team/sofia.jpg', qr: 'https://...' },
  { name: 'Ethan Brooks', role: 'Tech Sales', photo: '/team/ethan.jpg', qr: 'https://...' }
]" />
```

**Constraint:** exactly 4 people. With ≠ 4, the layout still renders but visually breaks the POTX intent. v0.4.1 emits a `console.warn` during dev when `people.length !== 4` (no build error — authors of 3-person teams need it to render rather than throw).

**Title:** the `Bio` component does not render its own title. The host slide (`layout: title-only` is the POTX convention) provides the title. v0.4.1 will rebuild slide 14's title rendering as part of the team-mode component to match POTX's white-title-overlay-on-band — see §5.2.

## 6. AnvilGridDecoration tile period adjustment

POTX uses what looks like the same anvil silhouette but at an apparently different tile size than v0.4.0's 60×30px. Confirming whether the existing v0.4.0 tile size matches POTX's density requires a side-by-side render comparison after Stage A lands. If the tile period needs adjusting:

- v0.4.0's `<AnvilGridDecoration>` exposes `bg` and `color` props but NOT a `tileSize` prop.
- v0.4.1 may need to add an optional `tileSize` prop (e.g., `'sm' | 'md' | 'lg'` mapping to `30×15`, `60×30`, `90×45`) so different layouts can pick the density that matches their POTX target.
- Defer this until visual-comparison Stage in the implementation phase confirms whether the existing tile is right.
