## 1. Summary

v0.4.2 ships six things:

1. **CI production-build smoke test** — appends `npm run build` to the existing visual-regression workflow, catching production-only failures (like v0.4.1's EISDIR/ENOENT) before they reach GitHub Pages.
2. **A-5: Divider rebuild** — all 4 POTX variants (a, b, c, d), each measured pixel-by-pixel.
3. **RipplePattern.vue removal** — gated on the divider rebuild (which was the last consumer apart from thank-you).
4. **A-8: Thank-you A** — minimal white close: contact-info label top-left, SAP+tagline lockup bottom-right, © legal line, page number.
5. **A-9: Thank-you B** — substantial close: top 50% sap-blue-10 anvil band, bottom 50% white with "Thank you." + contact card + QR + tagline + © line.
6. **A-6: Quote layout** — sap-blue-2 background with a giant flat-anvil-shaped highlight (sap-blue-4) behind quote text.
7. **A-7: Q&A layout** — sap-blue-2 background with photo + outlined Flat Anvil shape (sap-blue-6) as photo frame.

All values are measured from POTX renders, not estimated. Every dimension below is followed by `(POTX: x of W)` showing the source measurement.

## 2. Goals

1. **Faithfully reproduce 4 POTX divider variants** — every measurable value matches POTX within ±1% slide-relative tolerance.
2. **Faithfully reproduce POTX thank-you-a, thank-you-b, quote, qa** — same standard.
3. **Reuse existing primitives** — `<AnvilGridDecoration>`, `<FlatAnvil>`, `<ClassificationFooter>`. No new decoration components.
4. **Delete `RipplePattern.vue`** — the v0.3 wrong-abstraction component, finally gone after divider/thank-you migrate off it.
5. **Add CI safeguard** — `npm run build` runs on every PR so production-only failures don't reach GitHub Pages.
6. **Backward compatibility** — v0.3 author API for all four layouts continues to work; visual changes apply automatically.

## 3. Non-goals

- Cover variants `b`–`j` rebuild — still v0.4.3+ work (the audit's lowest-priority item; they're color variations of cover-a).
- Other audit findings (A-10 DataPoint, A-11 NumberedCards content layouts) — defer to v0.4.3+ as new content layouts.
- Full visual-regression baseline refresh — still gated on the upstream `bi/o` icon-resolution flake.
- Internationalization — `en-US` only.
