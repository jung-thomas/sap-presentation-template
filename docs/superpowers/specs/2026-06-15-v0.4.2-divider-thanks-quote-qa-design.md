## 1. Summary

v0.4.2 ships six things:

1. **CI production-build smoke test** — appends `npm run build` to the existing visual-regression workflow, catching production-only failures (like v0.4.1's EISDIR/ENOENT) before they reach GitHub Pages.
2. **A-5: Divider rebuild** — all 4 POTX variants (a, b, c, d), each measured pixel-by-pixel.
3. **RipplePattern.vue removal** — gated on the divider rebuild (which was the last consumer apart from thank-you).
4. **A-8: Thank-you A** — minimal white close: contact-info label top-left, SAP+tagline lockup bottom-right, © legal line, page number.
5. **A-9: Thank-you B** — substantial close: top 50% sap-blue-10 anvil band, bottom 50% white with "Thank you." + contact card + QR + tagline + © line.
6. **A-6: Quote layout** — sap-blue-2 background with a giant flat-anvil-shaped highlight (sap-blue-4) behind quote text.
7. **A-7: Q&A layout** — sap-blue-2 background with photo + outlined Flat Anvil shape (sap-blue-6) as photo frame.

All values are measured from POTX renders, not estimated. Every dimension below is followed by `(POTX: x of W)` showing the source measurement.

## 2. Goals

1. **Faithfully reproduce 4 POTX divider variants** — every measurable value matches POTX within ±1% slide-relative tolerance.
2. **Faithfully reproduce POTX thank-you-a, thank-you-b, quote, qa** — same standard.
3. **Reuse existing primitives** — `<AnvilGridDecoration>`, `<FlatAnvil>`, `<ClassificationFooter>`. No new decoration components.
4. **Delete `RipplePattern.vue`** — the v0.3 wrong-abstraction component, finally gone after divider/thank-you migrate off it.
5. **Add CI safeguard** — `npm run build` runs on every PR so production-only failures don't reach GitHub Pages.
6. **Backward compatibility** — v0.3 author API for all four layouts continues to work; visual changes apply automatically.

## 3. Non-goals

- Cover variants `b`–`j` rebuild — still v0.4.3+ work (the audit's lowest-priority item; they're color variations of cover-a).
- Other audit findings (A-10 DataPoint, A-11 NumberedCards content layouts) — defer to v0.4.3+ as new content layouts.
- Full visual-regression baseline refresh — still gated on the upstream `bi/o` icon-resolution flake.
- Internationalization — `en-US` only.

## 4. Divider rebuild (4 variants)

### 4.1 POTX measurements (all 1920×1080)

All four divider variants share the same general shape: SAP logo top-left, title centered or left-aligned in the upper portion, classification footer + page number bottom. The decoration changes per variant.

| Variant | Background | Decoration | Text color |
|---|---|---|---|
| **a** | `#ffffff` (white) | None — minimal | `var(--sap-text-primary)` (black) |
| **b** | `#1B90FF` (sap-blue-6) | Multiple flat-anvil shapes overlay (sap-blue-7 + sap-blue-10 + sap-blue-2 accents) | white |
| **c** | Top half white, bottom 50.2% `#002A86` (sap-blue-10) with `#1B90FF` (sap-blue-6) anvil tile pattern | `<AnvilGridDecoration>` constrained to bottom half | white-on-blue half + black-on-white half |
| **d** | White + horizontal `#D1EFFF` (sap-blue-2) band y=280-805 (≈ 49% of slide height, vertically centered) | Pale-blue band only | black |

### 4.2 Architecture

**Single layout file** (`theme/layouts/divider.vue`) composing:

- A root `<div class="divider divider--<v>">` with v∈{a, b, c, d}
- `<img class="divider-logo">` — SAP logo (white svg on dark bg, primary svg on light bg)
- `<h1 class="divider-title">` — title text
- `<slot />` — body content (e.g., section number)
- `<ClassificationFooter />` — bottom-left classification

Per-variant decorations:

- **`a`**: no decoration, all white
- **`b`**: `<FlatAnvil>` shapes positioned as multi-shape composition (3-4 instances at different positions/colors)
- **`c`**: `<AnvilGridDecoration>` constrained to bottom 50%, `bg=var(--sap-blue-10)` `color=var(--sap-blue-6)`
- **`d`**: a positioned `<div class="divider-band">` for the horizontal blue-2 band (no `<AnvilGridDecoration>` — just a solid pale-blue band; no anvil pattern)

Title positioning per variant follows POTX:
- a, b, d: title vertically centered (top: 40% range)
- c: title in the white top half, anchored at top: 25% (so it doesn't fall on the blue/anvil bottom)

### 4.3 Author API

**Unchanged from v0.3:**

```yaml
---
layout: divider
title: Section Title
variant: a   # 'a' | 'b' | 'c' | 'd'  (default 'a')
classification: PUBLIC
---

Optional body content (e.g., "Part 1 of 6")
```

No new fields. Existing decks render unchanged in shape; only the visual treatment per variant updates to match POTX.

### 4.4 RipplePattern.vue removal

After divider migrates to `<AnvilGridDecoration>` (variant c) and `<FlatAnvil>` (variant b), and after thank-you migrates (§6), `RipplePattern.vue` is the only remaining consumer of itself. Delete the file:

- `git rm theme/components/decorations/RipplePattern.vue`
- Any imports of it must already be gone after Tasks 4–6 land

This closes the v0.3 wrong-abstraction loop opened in v0.4.0 (where I tried to delete it and reverted).

## 5. Thank-you A (minimal close)

### 5.1 POTX measurements (thank-you-a.png, 1920×1080)

| Element | Position | Notes |
|---|---|---|
| Slide canvas | 1920 × 1080 | All white background |
| "Contact information:" label | y=464–488 (h=25px) | small text, top-left |
| SAP logo + "Bring out your best." tagline | y=944–999 (h=56px) | bottom-right |
| © legal notice line | y=1034–1046 (h=13px) | bottom-center |
| Page number | y=1037–1045 (h≈9px) | bottom-right corner |

The slide is intentionally sparse. This is the SAP corporate close pattern: brand lockup + legal line + page number, nothing else.

### 5.2 Architecture

Single layout file shape:

- Root `<div class="thank-you thank-you--a">` — white background
- "Contact information:" label rendered at `top: 43%; left: 4.2%` (POTX-derived from y=464/1080)
- `<SapTaglineLockup />` component anchored bottom-right at `right: 4.2%; bottom: ~7.5%`
- `<LegalNotice />` component anchored bottom-center
- `<ClassificationFooter />` already handles the page-number role

`<SapTaglineLockup>` is a new small component (~30 lines) that renders the SAP logo (primary color) + "Bring out your best." text in 72-Bold. Reusable across thank-you-a, thank-you-b, and possibly future closing slides.

`<LegalNotice>` is a new small component that renders the © legal-notice text. Centralized so the legal team can update one file.

### 5.3 Author API

```yaml
---
layout: thank-you
variant: a
classification: PUBLIC
---

# Optional body content (e.g., contact details)
```

The variant key dispatches between A and B. Default is `a` for v0.4.2 (was `b` in v0.3, but `a` is the simpler default).

### 6.2 Architecture

Same layout file, dispatches per variant. For `b`:

- Root `<div class="thank-you thank-you--b">`
- `<AnvilGridDecoration class="thanks-band" bg="var(--sap-blue-10)" color="var(--sap-blue-6)" />` constrained to top 50%
- "Thank you." `<h1>` at `top: 60%; left: 4.2%` (POTX y=647/1080)
- Contact-card `<div>` at `top: 65%; right: 25%` containing Name / Title / Address / City / Email lines (sourced from frontmatter)
- `<QRCode>` at `top: 65%; right: 4.2%; width: 7%` (POTX QR is square, ~110px on 1920)
- `<SapTaglineLockup />` at `bottom: 7%; right: 18%`
- `<LegalNotice />` at `bottom: 4%; left: 50%; transform: translateX(-50%)`

### 6.3 Author API

```yaml
---
layout: thank-you
variant: b
presenter: thomas-jung   # resolves contact details
qrUrl: https://people.sap.com/thomas
classification: PUBLIC
---
```

Contact details are sourced from `presenters/<slug>.yaml` (existing fixture). New optional fields on the presenter file: `address`, `city`, `email`. If absent, those rows of the contact card render empty.

`qrUrl:` is a new optional field. If absent, the QR position renders empty (no broken QR).

## 7. Quote layout

### 7.1 POTX measurements (quote.png, 1920×1080)

| Element | Value |
|---|---|
| Background | `#D1EFFF` (sap-blue-2) |
| Flat-anvil highlight color | `#89D1FF` (sap-blue-4) — measured 19,688 pixels in the slide |
| Quote text color | `#000000` (pure black) |
| "Quote" eyebrow label | tiny, top-left of slide |
| Speaker / Company / Role line | bottom-right, small grey |
| Page number | bottom-right corner |

The visual signature is the **giant flat-anvil-shaped highlight in sap-blue-4 sitting behind the quote text**, with the rest of the slide on the lighter sap-blue-2 background. Approximate size: takes up the central ~50% of the slide horizontally and the central ~60% vertically, with the anvil's diagonal cut creating visual asymmetry.

### 7.2 Architecture

- Root `<div class="layout quote">` — bg `var(--sap-blue-2)`
- Tiny "Quote" eyebrow at `top: 4%; left: 4.2%` in 72-Bold black, ~10px font
- `<FlatAnvil class="quote-highlight" color="var(--sap-blue-4)" />` positioned absolutely behind the quote text, taking up roughly the central 50%×60% of the slide (CSS `inset: 20% 25%`)
- `<blockquote class="quote-text">` with the quote content as `<slot />`, color black, font-size large (POTX shows ~36px on 1080-tall slide ≈ 3.3%)
- Speaker attribution `<footer>` bottom-right with three lines (name, company, role) in small grey
- `<ClassificationFooter />` bottom-left

### 7.3 Author API

```yaml
---
layout: quote
author: Steve Lucas
company: SAP
role: CEO
classification: PUBLIC
---

> Innovation is the lifeblood of our success.
```

`author` and `source` from v0.3 still work (`source` aliases `company` for backward compat). New `role:` field is additive (renders as third line if present, suppressed otherwise).

## 8. Q&A layout

### 8.1 POTX measurements (qa.png, 1920×1080)

| Element | Value |
|---|---|
| Background | `#D1EFFF` (sap-blue-2) — same as Quote |
| Outlined Flat Anvil shape (right-half photo frame) | `#1B90FF` (sap-blue-6) — measured 1,417 pixels (thin outline, not filled) |
| Eyebrow text "Your perspective matters" | top-left, small black |
| Title "Join the conversation" | large bold black, top-left below eyebrow |
| Subtitle "You ask. We answer." | medium black, below title |
| Photo placement | inside the outlined Flat Anvil shape, right half of slide |

The visual signature is the **outlined (not filled) Flat Anvil shape on the right half of the slide, framing a photo of the speaker / panel**. Only ~1,417 px of bright sap-blue-6 → it's a stroke outline of the FlatAnvil silhouette, not a fill.

### 8.2 Architecture

- Root `<div class="layout qa">` — bg `var(--sap-blue-2)`
- Left-half content region: eyebrow + title + subtitle + presenter compact card
- Right-half decoration:
  - A new `<FlatAnvilOutline>` component (or a CSS-stroked `<FlatAnvil>` variant) — renders the canonical anvil silhouette as a stroke (no fill), color `var(--sap-blue-6)`, stroke-width ~6px on a 1920-wide slide
  - `<PhotoFrame placement="right-half">` (existing v0.4.0 component) for the speaker photo inside the anvil outline

The new `<FlatAnvilOutline>` is small (~20 lines): same SVG path as `<FlatAnvil>` but with `fill="none" stroke="currentColor" stroke-width="6"` instead. Reusable for any future "outlined anvil" treatment.

### 8.3 Author API

```yaml
---
layout: q-and-a
title: Join the conversation
eyebrow: Your perspective matters
subtitle: You ask. We answer.
presenter: thomas-jung
image: /covers/qa-photo.png
classification: PUBLIC
---
```

Fields:
- `eyebrow` — new optional, small text above title
- `title` — main heading (defaults to "Questions?" for backward compat)
- `subtitle` — new optional, medium text below title
- `presenter` — unchanged, renders compact bio
- `image` — new optional, photo for the right-half outlined-anvil frame
