> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the Agenda layout and the Bio team-mode component to faithfully reproduce POTX agenda-b and POTX slide 14 ("Meet our team"), using pixel-measured values from the audit's POTX renders.

## How to use this plan

Execute tasks in order. Each task has a series of `- [ ]` checkboxes; check them off as you complete each step. Some tasks build on prior tasks (e.g., Task 5 needs Task 4 done first). Stage boundaries (A/B/C/D) are commit-friendly checkpoints — a partial release that ends at any stage boundary is shippable, just not feature-complete.

**Conventions:**

- All code blocks are the literal contents to write or paste (no `// ...` placeholders).
- All commands are exact; expected output is described in plain English.
- Commit messages are conventional-commits format with body explaining the why where helpful.
- TDD: red → green → refactor → commit. Where logic isn't testable in unit tests (template-only Vue components, scoped CSS), the plan documents that and defers verification to visual regression in Stage D.
- File paths are relative to the worktree root (`d:\projects\presentation-template\.claude\worktrees\v0.4.1-hotfixes`).
