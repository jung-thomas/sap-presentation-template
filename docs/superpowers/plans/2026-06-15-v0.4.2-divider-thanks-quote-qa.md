# v0.4.2 — Divider, Thank-you, Quote, Q&A Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild divider (4 variants), thank-you (A and B), quote, and Q&A layouts to faithfully reproduce POTX renders, plus add a `npm run build` CI smoke test and finally remove `RipplePattern.vue`.

**Architecture:** All layouts compose existing v0.4.0/v0.4.1 primitives (`<AnvilGridDecoration>`, `<FlatAnvil>`) plus three new small components (`<SapTaglineLockup>`, `<LegalNotice>`, `<FlatAnvilOutline>`). RipplePattern.vue is deleted only after both divider and thank-you migrate off it (strict task ordering).

**Tech Stack:** Node ≥22 · TypeScript · Vue 3 · Vite · Slidev 0.49 · Vitest · Playwright

**Spec:** [docs/superpowers/specs/2026-06-15-v0.4.2-divider-thanks-quote-qa-design.md](../specs/2026-06-15-v0.4.2-divider-thanks-quote-qa-design.md)

## How to use this plan

Execute tasks in order; some tasks are parallel-safe within a stage but stage boundaries are sequential. Each task has `- [ ]` checkbox steps; check them off as you complete each. Stage boundaries are commit-friendly checkpoints.

**Conventions:**

- All code blocks are the literal contents to write or paste (no `// ...` placeholders).
- Commands are exact; expected output described in plain English.
- Conventional-commits messages with body explaining the why.
- TDD: red → green → refactor → commit. Where logic isn't testable in unit tests (template-only Vue, scoped CSS), the plan documents that and defers verification to manual sanity in Stage F.
